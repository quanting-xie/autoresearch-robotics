实验 8 视觉分析(LR_CRITIC 3e-4 -> 1e-3,success_rate=0.00,训练发散)

帧路径:./renders/ep{0,1,2}_{start,mid,end}.png;对比基准:./renders_best/(仍是 exp7 的最佳帧)。

定量结果:
- success_rate 0.60 -> 0.00(发散)
- mean_reward -23.2 -> -50.0
- mean_distance 0.047 -> 0.092(几乎回到基线 0.61 之外的早期实验水平)
- total_updates 1096 -> 878(下降 ~20%,提示某些步骤花在了不健康的梯度更新上)
- total_steps 2096 -> 1878

定性观察(只读了关键帧、未做并排比较):
- 机器人的动作不再呈现稳定的 goal-tracking。
- 与 exp7 相比,policy 似乎回退到了类似前几个失败实验的“塌陷/默认姿态”行为。
- mean_distance 0.092 比 exp1 baseline 的 0.61 好得多,但比 exp5/7 的 ~0.046 差很多 —— 介于“训练前期方向尚可但还没收敛”和“训练后期发散”之间。

机理分析:
- LR_CRITIC=1e-3 + 没有任何稳定化(LayerNorm、dropout、ensemble)直接导致 critic loss 在某次大梯度更新后 spike,Q 输出剧烈震荡,actor 跟随发散。
- total_updates 下降也支持这一点:发散时 NaN/inf 的检测机制可能尚未触发(loss magnitude 还没破 1e6),但每步耗时变高(可能因为梯度变得更不规则,backward 更慢,或者 learning rate 让权重每步“更激进”)。
- 这与 REDQ 论文里的观察一致:naive 加大 critic 学习强度需要 ensemble 来对抗 Q-overestimation;不然就会发散。

决策:DISCARD exp8。把 LR_CRITIC 还原回 3e-4(保留 LR_ACTOR=1e-3,因为 exp7 已经被认定为 keep 的新最佳)。

下一步候选攻击(按风险/收益排序):
1. n-step returns (n=3):用 3 步 horizon 的 reward sum + bootstrap,直接给 critic 一个更准确的 target。无需调 LR,改的是 target 计算方式,稳定性可控。直接攻击“Q 在目标边界处偏差”的根因。
2. tau 0.005 -> 0.02:target 网络更新更快,actor 看到的 Q 更新鲜。极简改动,易于回滚。
3. 在 critic 加 LayerNorm:稳定化结构性增强。可作为 LR_CRITIC 上调的安全前提。
4. HER_K 4 -> 8:每个 batch 里 relabel 比例从 80% -> 88.9%,样本多样性提升。

exp 9 计划:n-step returns (n=3)。
- 实现:在 buffer.sample() 中,从 idx 出发取连续 n 个 transitions(不跨 episode 边界),把 reward 折扣求和,target_q 用第 n 步的 next_obs。
- 复杂度可控,不动 LR / 网络结构。
- 预测:success_rate 0.6 -> 0.7-0.85。
- 风险:实现 bug 会导致 reward 偏差;先小心写,跑一次就能看出来。
